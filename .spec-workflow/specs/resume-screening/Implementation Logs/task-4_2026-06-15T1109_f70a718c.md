# Implementation Log: Task 4

**Summary:** 实现 LLM 适配器层（Base Protocol + ZhipuAdapter GLM-4-Plus + QwenAdapter qwen-max + MockAdapter + LLMRouter）。BaseLLMAdapter 定义统一 chat 接口（messages/response_schema/temperature/timeout/model）。MockAdapter 支持注入 response_override / failures_before_success / name 覆盖 / cost_per_1k，作为测试与本地开发首选。Zhipu/Qwen 用 asyncio.to_thread 包装同步 SDK；schema 模式下追加 JSON 输出指令到 system prompt，再走 _json.extract_json + Pydantic 解析（容忍代码块、字符串内 {}、平衡括号）。LLMRouter 实现 scope-based 路由（默认 zhipu→qwen，可按 scope 覆盖）、熔断（5min 内 3 次失败冷却 5min）、重试（max_retries+1 次，base 200ms 指数退避）、schema 错误立刻切 fallback、llm_calls 表写入（best-effort，DB 失败不阻塞主流程）。全套 81 个测试通过（adapters 40 + models 12 + core 29）。
