# Implementation Log: Task 18

**Summary:** 实现 ReasoningService（推荐理由 + 淘汰理由 + 事实一致性校验）。核心三件套：(1) FactValidator 纯函数：字符串匹配 + 同义词词典（python→[python, py, django, flask, fastapi] 等），支持中英文混合关键词提取；(2) generate_recommend：调 LLM(scope='reasoning') 生成 3-5 条 bullet + evidence → 事实校验剔除无原文支持的 bullet → validated 标记 → upsert；(3) persist_disqualify：纯规则构造淘汰理由，直接基于 FilterService.reasons（已是 "<规则名>: <值> vs <要求>" 格式），不调 LLM。集成到 scorer_task.run_score：score 完成后自动查 ScreeningResult.disqualified 决定走 recommend 还是 disqualify；reasoning 失败不阻塞 score 已写入的结果。Prompt 设计：要求 LLM 给出 evidence 数组（每条 bullet 对应一个关键词），用于事后字符串匹配校验。同义词字典覆盖 python/java/go/rust/js/react/vue/fastapi/postgresql/redis/docker/kubernetes/微服务/学历/年限等常见术语。
